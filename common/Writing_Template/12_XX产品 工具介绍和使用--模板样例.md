# MindStudio Profiler Analyze

## 简介

MindStudio Profiler Analyze（msprof-analyze，MindStudio性能分析工具）是MindStudio全流程工具链推出的性能分析工具，基于采集的性能数据进行分析，识别AI作业中的性能瓶颈。

**基本概念**

- msprof-analyze：是mstt工具针对训练&大模型场景，提供端到端的性能调优工具。

## 使用前准备

**环境准备**

1. 准备一台基于Atlas 训练系列产品的训练服务器，并[安装NPU驱动和固件](https://www.hiascend.com/document/detail/zh/canncommercial/82RC1/softwareinst/instg/instg_0005.html?Mode=PmIns&InstallType=local&OS=Debian&Software=cannToolKit)。

2. 安装msprof-analyze，命令如下：

   ```
   pip install msprof-analyze
   ```

   提示出现如下信息则表示安装成功。

   ```
   Successfully installed msprof-analyze-{version}
   ```

**约束**

CANN软件版本8.0RC1之前仅支持对text格式文件分析，8.0RC1及之后支持text、db格式的采集数据分析。

## 快速入门

**前提条件**

1. 安装msprof-analyze工具，执行如下命令。

   ```
   pip install msprof-analyze
   ```

   提示如下信息则表示安装成功。

   ```
   Successfully installed msprof-analyze-{version}
   ```

2. 数据准备。

   MindSpore场景：参见《[MindSpore Profiler](https://www.hiascend.com/document/detail/zh/mindstudio/81RC1/T&ITools/Profiling/atlasprofiling_16_0087.html)》采集MindSpore场景性能数据。

   PyTorch场景：参见《[Ascend PyTorch Profiler](https://gitee.com/link?target=https%3A%2F%2Fwww.hiascend.com%2Fdocument%2Fdetail%2Fzh%2Fmindstudio%2F81RC1%2FT%26ITools%2FProfiling%2Fatlasprofiling_16_0090.html)》采集PyTorch场景性能数据。

**操作步骤**

执行advisor命令进行数据分析。

```
msprof-analyze advisor all -d $HOME/profiling_data/
```

详细分析结果介绍请参见《[advisor](https://gitee.com/ascend/mstt/tree/master/profiler/msprof_analyze/advisor)》中的“[报告解析](https://gitee.com/ascend/mstt/tree/master/profiler/msprof_analyze/advisor#报告解析无标杆)”。

**使用示例**

1. 数据准备。

   MindSpore场景：参见《[MindSpore Profiler](https://www.hiascend.com/document/detail/zh/mindstudio/81RC1/T&ITools/Profiling/atlasprofiling_16_0087.html)》采集MindSpore场景性能数据。

   PyTorch场景：参见《[Ascend PyTorch Profiler](https://gitee.com/link?target=https%3A%2F%2Fwww.hiascend.com%2Fdocument%2Fdetail%2Fzh%2Fmindstudio%2F81RC1%2FT%26ITools%2FProfiling%2Fatlasprofiling_16_0090.html)》采集PyTorch场景性能数据。

2. 执行advisor总体性能瓶颈分析。

   ```
   msprof-analyze advisor all -d $HOME/profiling_data/
   ```

   -d参数指定的是步骤1中获取的性能数据文件路径。

   分析结果输出相关简略建议到执行终端中，并在命令执行目录下生成“mstt_advisor_{timestamp}.html”和“/log/mstt_advisor_{timestamp}.xlsx”文件，并打印结果回显，打印结果回显详细信息请参见[输出说明](#输出说明)，生成的结果文件详细介绍请参见[输出结果文件说明](#输出结果文件说明)，。

```ColdFusion
g_shift     : 4
alpha_ min  : 232
tkp_shift   : 8
```

| 字段       | 说明                                                         |
| ---------- | ------------------------------------------------------------ |
| f          | 后续迭代计数。                                               |
| g_shift    | 用来更新alpha的参数G的偏移。                                 |

| tkp_shift  | tkp_shift，token桶更新周期的偏移2的N次方，如配置为8，则为2的8次方。 |

2. 分别在GPU和NPU环境下的训练脚本（main.py文件）中添加工具，如下所示。

   完整代码请参见[PyTorch精度数据采集代码样例](#PyTorch精度数据采集代码样例)。

   添加以下接口前，建议注释前面步骤的非训练脚本接口。

   ```python
    23
    24 import torch_npu
    25 from torch_npu.contrib import transfer_to_npu
    26
   ...
   ```
   
   精度数据会占据一定的磁盘空间，可能存在磁盘写满导致服务器不可用的风险。精度数据所需空间跟模型的参数、采集开关配置、采集的迭代数量有较大关系，须用户自行保证落盘目录下的可用磁盘空间。
   
3. 执行训练脚本命令，工具会采集模型训练过程中的精度数据。

   ```

   ```

   日志打印出现如下信息即可手动停止模型训练查看采集数据，节省时间。

   ```ColdFusion
   *****************************************************************************                        msprobe ends successfully.                        *****************************************************************************
   ```

   详细dump结果请参见[dump结果文件介绍](#dump结果文件介绍)。

### dump结果文件介绍

训练结束后，工具将 dump 的数据保存在 dump_path 参数指定的目录下。目录结构示例如下：

```
├── dump_path
│   ├── step0
│   |   ├── rank0
│   |   │   ├── dump_tensor_data
|   |   |   |    ├── Tensor.permute.1.forward.pt
|   |   |   |    ├── Functional.linear.5.backward.output.pt    # 命名格式为{api_type}.{api_name}.{API调用次数}.{forward/backward}.{input/output}.{参数序号}, 其中，“参数序号”表示该API的第n个输入或输出，例如1，则为第一个参数，若该参数为list格式，则根据list继续排序，例如1.1，表示该API的第1个参数的第1个元素。
...
```

| 字段             | 说明                                                         |
| ---------------- | ------------------------------------------------------------ |
| rank             | 设备 ID，每张卡的数据保存在对应的 `rank{ID}` 目录下。非分布式场景下没有 rank ID，目录名称为 rank。 |
| dump_tensor_data | 保存采集到的张量数据。                                       |
| dump.json        | 保存API或Module前反向数据的统计量信息。包含dump数据的API名称或Module名称，各数据的dtype、 shape、max、min、mean、L2norm（L2范数，平方根）统计信息以及当配置summary_mode="md5"时的CRC-32数据。具体介绍可参考[dump.json文件说明](https://gitee.com/ascend/mstt/blob/master/debug/accuracy_tools/msprobe/docs/27.dump_json_instruction.md#1-PyTorch场景下的dump.json文件)。 |
| ...              |                                                              |
